📖 Introduction

Bienvenue sur l'API Voaray pour marchands

L'API Voaray permet aux marchands d'intégrer facilement les paiements Mobile Money (MVola, Orange Money, Airtel Money) dans leurs applications et sites e-commerce.


Caractéristiques :

  • ✅ Intégration simple via clé API
  • ✅ Paiements sécurisés avec signature HMAC
  • ✅ Webhooks pour notifications en temps réel
  • ✅ Dashboard de statistiques
  • ✅ Support multi-opérateurs (MVola, Orange, Airtel)

🔐 Authentification

Comment authentifier vos requêtes API

Pour accéder aux endpoints protégés, vous devez inclure vos identifiants API dans les headers de la requête :

X-API-Key: votre_api_key
X-API-Secret: votre_api_secret

Ces identifiants vous sont fournis lors de la création de votre compte marchand.

⚠️ Important : Ne partagez jamais votre clé API et votre secret. Stockez-les de manière sécurisée.

📝 Créer un compte marchand

Inscription d'un nouveau marchand

POST /api/merchants/register
Crée un nouveau compte marchand et génère les clés API

Paramètres de la requête

Champ Type Requis Description
namestringRequisNom du marchand
emailstringRequisEmail du marchand (unique)
companystringOptionnelNom de l'entreprise
planstringOptionnelPlan (BASIC, PRO, ENTERPRISE) - défaut: BASIC
phonestringOptionnelNuméro de téléphone
webhookUrlstringOptionnelURL de webhook pour les notifications

Exemple de requête

curl -X POST http://localhost:5000/api/merchants/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mon E-commerce",
    "email": "contact@ecom.mg",
    "company": "Ecom MG SARL",
    "plan": "PRO",
    "phone": "0341234567",
    "webhookUrl": "https://ecom.mg/webhook"
  }'

Réponse succès (200)

{
  "success": true,
  "data": {
    "merchantId": "merch_123456",
    "name": "Mon E-commerce",
    "email": "contact@ecom.mg",
    "companyName": "Ecom MG SARL",
    "plan": "PRO",
    "apiKey": "voaray_abc123def456...",
    "apiSecret": "sec_789ghi012jkl...",
    "endpoints": {
      "createPayment": "http://localhost:5000/api/v1/payments",
      "getStatus": "http://localhost:5000/api/v1/payments/:id",
      "confirmPayment": "http://localhost:5000/api/v1/payments/confirm",
      "webhookUrl": "http://localhost:5000/api/webhooks/merchant"
    }
  }
}

✏️ Mettre à jour

Modifier les informations du marchand

ChampTypeDescription

📋 Transactions

Liste des transactions du marchand

GET /api/merchants/transactions
Retourne la liste paginée des transactions

Paramètres de requête

ChampTypeDescription
pagenumberNuméro de page (défaut: 1)
limitnumberÉléments par page (défaut: 50, max: 100)
statusstringFiltre par statut (PENDING, SUCCESS, FAILED)
startDatestringDate de début (ISO 8601)
endDatestringDate de fin (ISO 8601)

Exemple de requête

curl -X GET "http://localhost:5000/api/merchants/transactions?page=1&limit=10&status=SUCCESS" \
  -H "X-API-Key: voaray_abc123..." \
  -H "X-API-Secret: sec_789ghi..."

Réponse succès (200)

{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 150,
    "pages": 15
  }
}

🔄 Régénérer clé API

Générer une nouvelle clé API

POST /api/merchants/keys/:keyId/regenerate
Régénère une clé API existante

Exemple de requête

curl -X POST http://localhost:5000/api/merchants/keys/api_key_123/regenerate \
  -H "X-API-Key: voaray_abc123..." \
  -H "X-API-Secret: sec_789ghi..."

Réponse succès (200)

{
  "success": true,
  "data": {
    "key": "voaray_new_key_abc123...",
    "secret": "new_secret_xyz789...",
    "expiresAt": "2025-01-15T10:00:00Z"
  }
}
